## Data Flow Example

**User selects "Chicago Bulls" in Teams view:**

1. **Frontend** calls 6 API endpoints in parallel
2. **Backend** processes requests:
   - Queries `game_results` for ATS/OU stats
   - Queries `games` for next scheduled game
   - Queries `odds` for current market prices
   - Queries `odds_snapshots` for line movement history
   - Queries `player_game_stats` for minutes trends
   - Calculates EV and Kelly stakes
3. **Database** returns results
4. **Backend** transforms and sends JSON responses
5. **Frontend** renders 6 components with data
6. **User** sees complete betting decision screen in <3 seconds
